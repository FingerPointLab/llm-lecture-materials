### API Key
API 키는 클라이언트가 서버에 요청을 보낼 때, 해당 요청이 유효한 사용자로부터 왔는지 확인하는 데 사용됩니다. 이를 통해 서버는 요청을 인증하고, 요청자의 사용량을 추적하며, 권한이 있는 API만 접근하도록 제한할 수 있습니다. API 키는 고유 식별자로, 안전하게 보관하고 외부에 노출되지 않도록 주의해야 합니다.

### Webhook
Webhook은 특정 이벤트가 발생했을 때, 실시간으로 외부 서버에 알림을 보내는 방식입니다. 예를 들어, 결제 완료나 주문 상태 변경 같은 이벤트가 발생하면, 미리 지정된 URL로 POST 요청을 보내 정보를 전달합니다. Webhook은 폴링 없이도 즉각적인 반응이 가능해 시스템 간 연동에 효율적입니다.

### OAuth
OAuth는 사용자의 로그인 정보를 공유하지 않고도 외부 애플리케이션이 사용자 계정에 접근할 수 있도록 하는 인증 프로토콜입니다. 사용자는 애플리케이션이 접근할 수 있는 권한 범위를 승인하며, 이를 통해 안전하고 제한적인 API 접근이 가능합니다. Google, Facebook 로그인 등에 널리 사용됩니다.

### REST API
REST API는 HTTP 기반의 설계 원칙을 따르는 API 스타일로, 자원을 URI로 표현하고, 자원에 대한 행위는 HTTP 메서드(GET, POST 등)로 수행합니다. 클라이언트-서버 구조를 유지하면서도 상태 정보를 서버에 저장하지 않는 stateless한 특징을 갖고 있어 확장성과 유연성이 뛰어납니다.

### JWT
JWT(Json Web Token)는 서버와 클라이언트 간 정보를 JSON 형식으로 안전하게 전달하기 위해 사용되는 토큰입니다. 주로 사용자 인증과 권한 부여에 활용되며, 토큰 내부에는 서명된 클레임이 포함되어 있어 위조 여부를 쉽게 검증할 수 있습니다. 로그인 유지, API 인증 등에 적합합니다.

### ERP (Enterprise Resource Planning)
생산과 회계, 구매, 재고 등 다양한 기능을 포함하는 ERP는 기업의 자원(인적, 물적, 재무적 자원 등)을 통합적으로 관리하기 위한 시스템입니다.

### SAP 생산오더
SAP 시스템에서는 CO01 트랜잭션 코드를 통해 생산오더를 생성합니다. 생산오더는 제조 현장에서 실제로 생산이 이루어질 수 있도록 지시하는 문서로, 자재번호, 수량, 생산지 공장 등의 정보를 포함합니다. 이를 통해 자재 소요계획(MRP)과 연결된 실제 생산 활동이 시작됩니다.

### SAP 자재이동
SAP에서 자재이동은 입고, 출고, 재고 이동 등의 프로세스를 포함하며, 대표적으로 MIGO 트랜잭션을 통해 처리됩니다. 자재이동 문서에는 이동 유형, 관련 문서 번호, 수량 등의 세부 정보가 포함되어 물류 흐름을 기록하고 추적할 수 있게 해줍니다.

### SAP 자재마스터
SAP 자재마스터(Material Master)는 제품이나 부품 등 자재에 대한 기술적, 구매적, 재무적 정보를 통합 관리하는 핵심 데이터입니다. 자재마스터는 MM01 트랜잭션을 통해 생성되며, 이는 구매, 생산, 재고 관리, 회계 등의 모듈에서 공통적으로 사용됩니다.

### SAP 회계전표
SAP의 회계전표는 재무 거래가 발생했을 때 생성되며, BKPF와 BSEG 테이블에 저장됩니다. 회계전표는 거래일, 계정번호, 금액, 회사코드 등 다양한 항목을 포함하며, 기업의 재무 상태를 실시간으로 기록하고 분석하는 데 중요한 역할을 합니다.

### SAP BOM
BOM(Bill of Materials)은 제품을 구성하는 모든 부품과 자재의 목록을 계층적으로 보여주는 문서입니다. SAP에서는 CS01 트랜잭션을 통해 BOM을 생성할 수 있으며, 이를 통해 제품 단가 계산, 생산계획 수립, 자재 소요 분석 등이 가능해집니다.

### 전자 계약의 해지 방법
전자 계약으로 체결된 보험은 대면 계약과 동일한 효력을 가지며, 해지 또한 온라인을 통해 간편하게 처리할 수 있습니다. 대부분의 보험사는 모바일 앱이나 홈페이지에서 로그인 후 ‘계약관리 > 해지 신청’ 메뉴를 통해 해지 절차를 진행할 수 있도록 시스템을 제공합니다. 해지 신청 시 본인 인증이 필요하며, 신분증 제출 대신 휴대폰 인증, 공동인증서, 간편인증 등의 수단이 사용됩니다. 해지 후 해지환급금은 등록된 계좌로 입금되며, 평균적으로 3~5영업일 이내에 처리가 완료됩니다. 단, 특정 상품은 온라인 해지가 제한되거나, 콜센터 상담 후 해지 가능한 경우도 있어 사전 확인이 필요합니다.

### 보험 계약해지
보험 계약을 해지하고자 할 경우, 계약자는 해지 사유서를 작성하고 신분증 사본과 함께 보험사에 제출해야 합니다. 일반적으로 해지 시점에 따라 해지환급금이 정해지며, 해지 사유가 불명확하거나 특별한 경우에는 별도의 심사가 이루어질 수 있습니다.

### 보험 계약해지 - 중도해지로 인한 불이익
보험 계약을 만기 이전에 중도 해지할 경우, 해지환급금은 납입 보험료보다 적을 수 있으며, 일부 보장형 상품의 경우 해지환급금이 전혀 없을 수도 있습니다. 특히 계약 초기에 해지하면 사업비가 공제되어 환급금이 거의 없거나 '0원'일 수 있으므로, 해지를 결정하기 전에 충분한 상담이 필요합니다.

### 보험 계약해지 - 단체보험의 해지 절차
단체보험의 경우, 계약자는 일반적으로 회사나 단체이며, 피보험자는 구성원 개인입니다. 이때 계약 해지를 원할 경우 단체 계약자가 전체 계약에 대해 해지 의사를 밝히고, 보험사와 협의를 거쳐 절차를 진행해야 합니다. 피보험자 개개인이 별도로 해지 신청을 할 수 없는 구조이기 때문에, 단체 계약 조건에 따른 제한이 있을 수 있습니다.

### 보험 계약해지 후 재가입 제한
일부 보험 상품은 계약 해지 이후 일정 기간 동안 동일 상품에 재가입하는 것을 제한하고 있습니다. 이러한 제한은 보험 가입 남용이나 반복 해지를 방지하기 위한 조치로, 보통 1개월에서 6개월까지의 대기 기간이 설정됩니다. 특히 고위험 질병 보장 보험의 경우, 해지 이후 바로 재가입하려 할 때 과거 해지 이력이 심사에 반영되어 거절될 수 있습니다. 또한 동일한 계약조건으로 재가입하더라도 보험료가 인상되거나 보장 내용이 축소될 수 있어, 계약 해지 전에 충분한 비교와 상담이 필요합니다.

### 보험 청약철회
청약철회는 계약자가 보험계약 체결 후 일정 기간 내에 별도의 불이익 없이 계약을 취소할 수 있는 제도입니다. 대체로 계약일로부터 15일 이내에 서면으로 청약철회 신청서를 제출하면 계약은 무효 처리됩니다. 이 경우 이미 납입한 보험료는 전액 환급됩니다.

### 보험금 청구
보험금 청구는 사고 발생 시 보장받기 위한 절차로, 사고 내용에 대한 증빙서류와 함께 청구서를 제출해야 합니다. 교통사고, 질병, 사망 등 다양한 사고 유형에 따라 요구되는 서류가 달라지며, 심사 후 지급 여부가 결정됩니다.

### 보험 계약대출
보험 계약대출은 해지환급금 범위 내에서 보험사가 일정 금액을 계약자에게 대출해주는 제도입니다. 대출금에는 이자가 적용되며, 보험 계약이 유지되는 동안에도 상환이 이루어질 수 있습니다. 만약 대출금이 해지환급금을 초과하면 계약이 실효될 수 있습니다.

### 보험 실효
